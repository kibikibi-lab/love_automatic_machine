## P-01: 田中 蓮（マッチングアプリ初心者）

```
年齢: 25歳
職業: 会社員（メーカー勤務）
マッチングアプリ歴: 3ヶ月
マッチング数: 月5〜10件
課題: 最初のメッセージで何を書けばいいかわからず、
      マッチしても送れないまま期限切れになることが多い
```

**動機・目標**
- マッチした相手に最初のメッセージを送れるようになりたい
- 相手に合わせた自然な文章を書きたいが、センスがないと感じている
- とにかく「会う」という体験をしてみたい

**行動パターン**
- マッチしても数日間メッセージを送れずに悩む
- 友人に「何て送ればいい？」と相談することがある
- テンプレメッセージを使ったことがあるが、返信率が低かった

**本サービスとの関係**
- AIが生成したメッセージをそのままコピーして使う
- 会話ルートも全部AIに任せる
- 使えば使うほど自分で考えなくなる → **「人をダメにする」体験の主役**

## P-02: 佐藤 美咲（マッチングアプリ経験者）

```
年齢: 28歳
職業: 営業職
マッチングアプリ歴: 2年
マッチング数: 月20〜30件
課題: 会話は続くが、なかなか実際に会う約束まで進まない。
      毎回同じような会話の繰り返しに疲弊している
```

**動機・目標**
- 会話を効率化して「会う」という成果に直結させたい
- 毎回ゼロから考えるのが面倒になってきた
- 自分のキャラクターを活かしつつ、AIに戦略を立ててほしい

**行動パターン**
- キャラクター設定を細かく登録して自分専用AIを作り込む
- 会話ルートを参考にしながら、ゴールまでの最短経路を狙う
- 振り返りレポートで「どこで失敗したか」を分析する

**本サービスとの関係**
- 最初は自分でアレンジしながら使うが、徐々にAI依存が深まる
- 振り返りレポートを見るたびに「次もAIに頼ろう」と思うようになる
- **「便利さに慣れて自分で考えなくなる」体験の体現者**

## P-03: 鈴木 翔（データ重視ユーザー）

```
年齢: 27歳
職業: エンジニア
マッチングアプリ歴: 1年
マッチング数: 月10〜15件
課題: 感覚ではなくデータで恋愛を最適化したい。
      どのメッセージが効果的かを数値で把握したい
```

**動機・目標**
- 返信率・会えた率などの指標を継続的に改善したい
- 振り返りレポートで「どのパターンが有効か」を分析したい
- 舐めプスコアの推移をグラフで見たい（自分がどれだけ舐めプしているかを数値で楽しむ）

**行動パターン**
- 毎回の会話後に振り返りレポートを必ず確認する
- 複数のメッセージパターンを試してA/Bテスト的に比較する
- スコアの変化を記録して「最適化」を楽しむ

**本サービスとの関係**
- データを見るほどAIへの依存が深まる
- 「数値が改善されている」という事実が自己正当化になる
- **「合理化によって舐めプが深まっていく」体験の体現者**

## P-04: 山田 葵（競争心ユーザー・おまけ）

```
年齢: 24歳
職業: フリーランス（デザイナー）
マッチングアプリ歴: 6ヶ月
マッチング数: 月15〜20件
課題: 自分がどれだけ会えたかを誇示したい。
      自分のメッセージが他の人に使われることに優越感を感じる
```

**動機・目標**
- 指名ランキング1位を目指す（自分がどれだけ会えたかを誇示したい）
- 自分が生成したメッセージが多くの人にコピーされることで承認欲求を満たす
- 「最も指名された人」として認められたい

**行動パターン**
- 高品質なメッセージを生成するために試行錯誤を繰り返す
- 自分のメッセージのコピー数を毎日確認する
- ランキング上位を維持するために積極的に使い続ける

**本サービスとの関係**
- 頑張って磨いたメッセージをP-01・P-02が無思考でコピーして使う
- **「頑張る人が頑張らない人を養う」エコシステムの供給側**
- 承認欲求（指名数）を満たすために頑張るが、その成果が他人の思考力を奪う皮肉な構造
