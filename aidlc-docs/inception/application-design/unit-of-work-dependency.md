## 依存関係詳細

### Unit 1 → Unit 3（ペルソナ分析）
- **種別**: 同期REST呼び出し（任意）
- **内容**: メッセージ生成前にペルソナ分析結果を取得する
- **注意**: ペルソナ分析はオプション。Unit 3が未実行でもUnit 1は動作する

### Unit 1 → Unit 5（キャラクター設定）
- **種別**: 同期REST呼び出し
- **内容**: メッセージ生成前にユーザーのキャラクター設定を取得する

### Unit 1 → Unit 7（スコア更新）
- **種別**: 同期REST呼び出し
- **内容**: メッセージ生成後に舐めプ度（軸1）を+1する

### Unit 2 → Unit 6（実績記録）
- **種別**: 同期REST呼び出し
- **内容**: 会話終了時に実績データを記録する

### Unit 2 → Unit 7（スコア更新）
- **種別**: 同期REST呼び出し
- **内容**: 返信対応後に舐めプ度（軸1）を+1する

### Unit 5 → Unit 4（ユーザーID）
- **種別**: 同期REST呼び出し
- **内容**: キャラクター設定の保存・取得にユーザーIDを使用する

### Unit 6 → Unit 4（ユーザーID）
- **種別**: 同期REST呼び出し
- **内容**: 実績記録にユーザーIDを使用する

### Unit 7 → Unit 4（ユーザーID）
- **種別**: 同期REST呼び出し
- **内容**: スコア管理にユーザーIDを使用する

### Unit 8 → Unit 1〜7（全API）
- **種別**: HTTP REST（API Gateway経由）
- **内容**: フロントエンドが全バックエンドAPIを呼び出す

### Unit 9 → Unit 1〜8（インフラ定義）
- **種別**: CDKコード依存
- **内容**: 全UnitのAWSリソースをCDKで定義する

## 開発順序（推奨）

依存関係を考慮した実装順序：

```
フェーズ1（独立Unit・先に実装）:
  Unit 3: 相手ペルソナ分析エンジン（依存なし）
  Unit 4: ユーザー管理・認証（依存なし）

フェーズ2（フェーズ1に依存）:
  Unit 5: キャラクター設定管理（Unit 4に依存）
  Unit 7: 舐めプスコア・指名ランキング（Unit 4に依存）

フェーズ3（フェーズ2に依存）:
  Unit 1: AIメッセージ生成エンジン（Unit 3・5・7に依存）
  Unit 6: 実績トラッキング・レポート（Unit 4に依存）

フェーズ4（フェーズ3に依存）:
  Unit 2: 会話ルート設計エンジン（Unit 6・7に依存）

フェーズ5（全バックエンド完了後）:
  Unit 8: フロントエンド UI（Unit 1〜7に依存）
  Unit 9: AWSインフラ構成（Unit 1〜8に依存）
```
