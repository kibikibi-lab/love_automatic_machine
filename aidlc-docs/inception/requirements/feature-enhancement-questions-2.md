## Question 2 Unit構成の案

以下の8 Unit構成を提案します。
ドキュメント審査メインなので、粒度を細かくして「Unit分解の適切さ」で高評価を狙います。

```
Unit 1: AIメッセージ生成エンジン
  - 最初のメッセージ生成（Amazon Bedrock）
  - キャラクター設定 + 相手情報 + ゴールを組み合わせてパーソナライズ

Unit 2: 会話ルート設計エンジン
  - ゴールまでの会話シナリオ生成・返信予測
  - 返信が来たときの練り直し機能
  - 離脱ポイントの記録

Unit 3: 相手ペルソナ分析エンジン（新規・機能C）
  - 相手のプロフィール文をBedrockで分析
  - 人物像・性格傾向・刺さるアプローチを推定

Unit 4: ユーザー管理・認証
  - Amazon Cognitoによる認証
  - ユーザープロフィール管理

Unit 5: キャラクター設定管理
  - 話し方・趣味・成功パターンの登録・更新
  - Unit 1・2のAI生成に反映

Unit 6: 実績トラッキング・レポート（新規・機能E）
  - 返信率・離脱ポイント・会えた実績の記録
  - 会話終了後の振り返りレポート自動生成

Unit 7: 人間力崩壊スコア・ランキング（新規・機能A）
  - 自分の人間力劣化スコア（使うたびにマイナス）
  - 他人の人間力破壊スコア（コピーされるたびにマイナス）
  - 匿名ランキング（最もダメになった人が1位）
  - ゲーミフィケーション（バッジ・称号）

Unit 8: フロントエンド UI
  - Next.js（TypeScript）
  - 「ダメになっていく」演出UI
  - 各Unit APIとの連携
```

この構成についてどう思いますか？

A) この8 Unit構成でOK
B) 一部変更したい（[Answer]: の後に変更内容を記載してください）
C) もっとシンプルにしたい（5〜6 Unit程度）
D) Other (please describe after [Answer]: tag below)
